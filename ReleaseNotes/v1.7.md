## What's new

- **Sign in with GitHub Device Flow.** StarLight no longer relies on a redirect callback to authorize your account. Click *Sign in with GitHub*, and the app shows a short user code, copies it to your clipboard, and opens [github.com/login/device](https://github.com/login/device). Paste the code in your browser to authorize.
- **Multi-device friendly.** Every device now holds its own independent token. Signing in on a second machine no longer evicts the first.
- **No more client secret in the binary.** The previous Web Flow required an OAuth client secret embedded inside the `.app`. Device Flow does not, so the binary you download from this page is one less secret in the world.
- **New Settings entry: *Manage Authorizations on GitHub***, a one-click deep link to the page where you can review or revoke this app's authorization.

## Migration notes

- Existing keychain tokens from v1.6 and earlier remain valid; **you do not need to sign in again after updating**.
- Next time you sign out and back in, the new Device Flow UI takes over.
